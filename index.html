A tasklist

  1. útfæra onmousedown og að teikna á canvasinn (any shape) HKH
  2. create shapes classes ASS
  3. add all the shapes to an array HKH
  4. add color selector - http://jweir.github.com/colorwheel/
  5. be able to add primitive drawing objects to the drawing
    • a circle HKH
    • rectangle HKH
    • line HKH
    • text
    • freehand drawing by a pen HKH
  6. all elements should be movable. Preferably it should also be possible to move many objects simultaneously
  7. should be possible to group these primitives together into a template, which can be saved and then added to a new whiteboard with ease
  8. should be possible to save and load a drawing

The requirement list is not written in stone, but the following is a list of possible features:

  • it should be possible to add primitive drawing objects to the drawing, such as a circle, rectangle, line, text and a pen (i.e. a freehand drawing)
  • It should be possible to group these primitives together into a template, which can be saved and then added to a new whiteboard with ease (example: Luca might want to create a "binary tree node" template, containing a circle with text inside, and two arrows pointing downwards from the circle. Halldór might perhaps want to create a "2nd order equation" template etc.)
    • all elements should be movable. Preferably it should also be possible to move many objects simultaneously (i.e. select many objects first, then move them all together) the application should support undo (and preferably redo as well)
  • it should be possible to save and load a drawing
  • Bonus points: the application should support math formulas, and symbols commonly used in math (such as Pi, the sum symbol etc.). You are free to experiment with the implementation of this, one possibility is to be able to write equations in some sort of a language which would then be converted to a drawing by the application.
